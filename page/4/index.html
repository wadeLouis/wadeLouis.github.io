Simulation 脚本结构

脚本是基于Scala类，包含四个部分： * HTTP 协议配置 * 请求头定义 * 测试方案定义 * 测试策略定义

HTTP协议配置

最基本的配置是baseURL属性，这个地址将会作为每个测试方案测试时的http地址的前缀。除了这个，还可以配置通用的请求头属性，这里的这个请求头在每个请求都有效。 实例：

val httpConf = http
  .baseURL("http://computer-database.gatling.io") // 主路径
  .acceptHeader("text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8") //通用头
  .doNotTrackHeader("1") // 不跟踪
  .acceptLanguageHeader("en-US,en;q=0.5") // 头语言
  .acceptEncodingHeader("gzip, deflate") // 编码方式
  .userAgentHeader("Mozilla/5.0 (Macintosh; Intel Mac OS X 10.8; rv:16.0) Gecko/20100101 Firefox/16.0") // 用户代理

Gatling 一步一步进阶 2 - 配置

2016-12-08
软件测试 性能测试 Gatling

Gatling 目录结构

  • bin 保存Gatling执行脚本和录制工具的脚本
  • conf 保存Gatling, Akka and Logback的配置文件.
  • lib 保存Gatling的一些库文件
  • user-files:
    • simulations 保存测试脚本，以包名为文件夹方式保存
    • data 保存参数化需要用的测试数据.
    • bodies 测试请求的body内容的模板.
  • results 保存测试日志和测试报告.

Gatling 一步一步进阶 0 - 入门实例

2016-12-08
软件测试 性能测试 Gatling

下载与安装

  • 下载地址：

http://gatling.io/#/resources/download

  • 安装
  1. 解压文件
  2. 确保已经安装或者配置好JDK，最新版本的Gatling需要JDK8
  3. Gatling的启动脚本和maven插件需要配置JAVA_HOME环境变量。
  4. Gatling解压后就可以使用了，启动脚本在bin目录下，gatling.sh是启动测试，recorder.sh是启动录制

Gatling默认字符编码是UTF-8，你如果要修改，则需要改动下面几项： 1、 录制的时候要选择编码方式 2、 在gatling.conf配置中修改编码方式 3、保证你修改的文件和脚本的编码方式是正确的
