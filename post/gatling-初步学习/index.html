Gatling 一步一步进阶 0 - 入门实例

下载与安装

  • 下载地址：

http://gatling.io/#/resources/download

  • 安装
  1. 解压文件
  2. 确保已经安装或者配置好JDK，最新版本的Gatling需要JDK8
  3. Gatling的启动脚本和maven插件需要配置JAVA_HOME环境变量。
  4. Gatling解压后就可以使用了，启动脚本在bin目录下，gatling.sh是启动测试，recorder.sh是启动录制

Gatling默认字符编码是UTF-8，你如果要修改，则需要改动下面几项： 1、 录制的时候要选择编码方式 2、 在gatling.conf配置中修改编码方式 3、保证你修改的文件和脚本的编码方式是正确的

关于录制工具的使用

  • 运行bin目录下的recorder.sh
  • 修改好你想要的端口，在package字段写个包名，class Name字段写类名
  • 勾选你想要的选项
  • 在Filters中添加你不想录制下来的选项，即定义你的过滤条件，比如*\.css, .*\.js and .*\.ico等，有白名单whitelist和黑名单blacklist。
  • 点击start开始录制
  • 打开浏览器，修改浏览器的代理，开始访问你的网站进行相关的操作
  • 操作完成后点击stop
  • 脚本会在user-file目录下的simulations中，文件夹是包
  • 执行bin目录下的gatling.sh -s 包名.类名 既开始执行测试
  • 结果保存在result目录下

package oms_Interface  // 定义包，和java语言一样

// 下面是导入部分，1-4 是必须要的
import scala.concurrent.duration._  // 1

import io.gatling.core.Predef._ // 2
import io.gatling.http.Predef._ // 3
import io.gatling.jdbc.Predef._  // 4
import java.util.concurrent.ThreadLocalRandom  // 这是直接导入了一个java的包 ，Scala语言可以直接这样吗？ 好神奇
class uploadImg extends Simulation {  //定义一个类 必须继承Simulation  好吧，和java写法一样
    // id,username,password
    val feeder = csv("userID.csv").random  
    // 定义了一个读取csv数据的对象，读取方式是随机。Gatling支持读取json，Sitemap，Redis，csv，数据库的数据用来做参数化。也可以把一系列的值变成feeder对象。userID.csv要放在user-file目录下的data目录中
    // 实际最后的对象是一个Iterator[Map[String, T]]。
    // 读取方式：.
    // .queue    默认方式 顺序取值，取值超出后会报异常
    // .random   随机去取值
    // .circular 循环取值   

//定义一个OMS对象，这里把测试过程和scenario的初始化分离开
    object OMS {
        // 定义对象，包含一系列的动作，即这就是一个测试用例
        val upload  = exec(http("Upload")
				.post("/modify-icon.post") // 使用post方法，这里的参数会和baseURL拼接起来
				.headers(headers_0) // 请求头
				.formParam("uid", "6153") // 这里我们使用了post 表单方式，所以用formParam
				.formParam("type", "6")  // 具体的可以去[cheat-sheet](http://gatling.io/#/cheat-sheet/2.2.3)查看
	    		.formParam("imgType", ThreadLocalRandom.current.nextInt(1,2))
	    		.formParam("imgName", "aaaaaasdasdada6")
				.formUpload("image_file","/home/xxxx/a.png")  //这里是文件上传
				.check(status.is(200))
                // check 检查返回内容，详细的内容可以看官方的[cheat-sheet](http://gatling.io/#/cheat-sheet/2.2.3)
				.check(jsonPath("$.message.imgName").saveAs("imgName"))
				.check(jsonPath("$.status")
                                        .is("success"))
			).pause(5) //pause 是用来设定思考时间的
	val download = exec(http("Download")
				.get("/img.get") // get方法
				.queryParam("fn","${imgName}")
				.check(status.is(200))
				.check(header("Content-Type").is("application/x-download"))
				.check(header("Content-Disposition").is("attachment;filename=${imgName}"))
			).pause(1)
    }



    // 设置浏览器信息
    val httpConf = http
        .baseURL("http://192.168.1.114:8099/file/api-git")
    	.acceptHeader("text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8")
    	.doNotTrackHeader("1")
   	    .acceptLanguageHeader("zh-CN,zh;q=0.8,en;q=0.6")
    	.acceptEncodingHeader("gzip, deflate")
    	.userAgentHeader("Mozilla/5.0 (Macintosh; Intel Mac OS X 10.8; rv:16.0) Gecko/20100101 Firefox/16.0")

    // 自定义header的一些参数
    val headers_0 = Map("Accept" -> "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8")
    // 初始化一个scenario 对象，格式是scenario(名称)[一些动作，比如.exec(动作).feed(xxx)]
    val up = scenario("Upload").exec(OMS.upload)
                //.feed(feeder)  因为测试用例和scenario对象分离开，所以feed写到测试用来里面好像找不到feeder这个对象了。

    val down = scenario("Down").exec(OMS.download)
    val up_down = scenario("up_down").exec(OMS.upload,OMS.download)
     // 设置请求次数 如下为在立即启动500个请求
    setUp(
             up_down.inject(atOnceUsers(500))
	    //down.inject(atOnceUsers(1)),
           // up.inject(atOnceUsers(1))
  	).protocols(httpConf)

}

上面就是基本的入门测试脚本，要写出这个脚本，只需要看看官方文档和Gatling安装目录user-file下自带的脚本就可以了。
