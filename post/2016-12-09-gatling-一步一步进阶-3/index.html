Simulation 脚本结构

脚本是基于Scala类，包含四个部分： * HTTP 协议配置 * 请求头定义 * 测试方案定义 * 测试策略定义

HTTP协议配置

最基本的配置是baseURL属性，这个地址将会作为每个测试方案测试时的http地址的前缀。除了这个，还可以配置通用的请求头属性，这里的这个请求头在每个请求都有效。 实例：

1
2
3
4
5
6
7

val httpConf = http
  .baseURL("http://computer-database.gatling.io") // 主路径
  .acceptHeader("text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8") //通用头
  .doNotTrackHeader("1") // 不跟踪
  .acceptLanguageHeader("en-US,en;q=0.5") // 头语言
  .acceptEncodingHeader("gzip, deflate") // 编码方式
  .userAgentHeader("Mozilla/5.0 (Macintosh; Intel Mac OS X 10.8; rv:16.0) Gecko/20100101 Firefox/16.0") // 用户代理

请求头定义

在HTTP协议配置时已经配置了通用的请求头，这个请求头在每个请求时都会使用。那针对特例的请求，可以添加自定义的请求头属性，写法如下：

1
2

val headersObject = Map(属性 -> 值)
// eg. val headers_10 = Map("Content-Type" -> """application/x-www-form-urlencoded""")

在特例的请求动作时配置http的属性 .headers(headersObject) 既可以。

测试方案定义

可以这样定义测试方案 val 方案对象 = scenario("方案名称") 。scenario对象两个基本方法是exec 和 pause，第一个是执行动作，第二个是思考时间。 在执行动作中定义http请求：

1
2
3
4

http("名称")
  .post("路径") // 请求方法：post  这个路径会和baseURL拼接起来
  .headers(headersObject) //自定义请求头
  .formParam("name", "Beautiful Computer") // 参数

# Hooks
提供两种方式：
* before 在测试之前执行的代码
* after 在测试之后执行的代码

两个方法都是针对整个测试脚本的
实例：
```python
before {
  println("Simulation is about to start!")
}

after {
  println("Simulation is finished!")
}

这两部分的内容不能使用gatling的专用语言，只能写基于scala的自定义的代码。
