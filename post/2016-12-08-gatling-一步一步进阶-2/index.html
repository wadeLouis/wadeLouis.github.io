Gatling 目录结构

  • bin 保存Gatling执行脚本和录制工具的脚本
  • conf 保存Gatling, Akka and Logback的配置文件.
  • lib 保存Gatling的一些库文件
  • user-files:
    • simulations 保存测试脚本，以包名为文件夹方式保存
    • data 保存参数化需要用的测试数据.
    • bodies 测试请求的body内容的模板.
  • results 保存测试日志和测试报告.

一些可配置项

JDK

JDK版本需要JDK8，如果需要增加一些JVM选项可以修改Gatling脚本，一下是默认的配置选项：

DEFAULT_JAVA_OPTS="-server"
DEFAULT_JAVA_OPTS="${DEFAULT_JAVA_OPTS} -Xmx1G"
DEFAULT_JAVA_OPTS="${DEFAULT_JAVA_OPTS} -XX:+UseG1GC -XX:MaxGCPauseMillis=30 -XX:G1HeapRegionSize=16m -XX:InitiatingHeapOccupancyPercent=75 -XX:+ParallelRefProcEnabled"
DEFAULT_JAVA_OPTS="${DEFAULT_JAVA_OPTS} -XX:+PerfDisableSharedMem -XX:+AggressiveOpts -XX:+OptimizeStringConcat"
DEFAULT_JAVA_OPTS="${DEFAULT_JAVA_OPTS} -XX:+HeapDumpOnOutOfMemoryError"
DEFAULT_JAVA_OPTS="${DEFAULT_JAVA_OPTS} -Djava.net.preferIPv4Stack=true -Djava.net.preferIPv6Addresses=false"
COMPILER_OPTS="-Xss100M $DEFAULT_JAVA_OPTS $JAVA_OPTS"

具体的可以根据需要修改。

IPv4 vs IPv6

默认是使用IPv4的，如果需要IPv6，修改启动脚本中这里：

1

DEFAULT_JAVA_OPTS="${DEFAULT_JAVA_OPTS} -Djava.net.preferIPv4Stack=true -Djava.net.preferIPv6Addresses=false"

系统配置

需要配置的是文件句柄ulimit和内核网络传输参数设置sysctl.conf 这些配置方法在百度很容易找到，具体的在这里不说了。 给几个链接： * http://gatling.io/docs/2.2.3/general/operations.html * [转] 设置ulimit值(Linux文件句柄数量)永久生效 * Ubuntu 12 ulimit 系统最大打开文件个数 设置 * 设置Linux内核参数 /etc/sysctl.conf * /etc/sysctl.conf 调优 & 优化Linux内核参数

Gatling 配置项

Gatling可以修改配置的地方有三个： * conf目录下的配置文件 * 命令行选项 * $JAVA_OPTS环境变量

配置文件

logback.xml

这个文件可以配置Gatling日志的级别，需要更多的配置可以查看这里 Gatling有一些专用的日志记录器logger：

<!-- Uncomment for logging ALL HTTP request and responses -->
	<!-- 	<logger name="io.gatling.http.ahc" level="TRACE" /> -->
	<!--    <logger name="io.gatling.http.response" level="TRACE" /> -->
	<!-- Uncomment for logging ONLY FAILED HTTP request and responses -->
	<!-- 	<logger name="io.gatling.http.ahc" level="DEBUG" /> -->
<!-- <logger name="io.gatling.http.response" level="DEBUG" /> -->

默认是不开启的，可以去掉注释打开日志记录。

gatling.conf

配置文件基于Typesafe Config library，使用 HOCON 格式 配置项的优先级如下： 系统环境变量>gatling.conf> gatling-defaults.conf gatling-defaults.conf建议不要修改，这是提供给gatling-core jar使用的 如果要改配置，可以修改gatling.conf文件或者设置系统变量(变量名称要符合 HOCON Path)
