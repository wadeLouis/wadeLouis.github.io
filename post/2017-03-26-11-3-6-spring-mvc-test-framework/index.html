11.3.6 Spring MVC Test Framework

2017-03-20
软件测试 单元测试 Junit Spring MVC test

文章目录

  • 服务端侧的测试

独立的项目 在包括 Spring Framework 3.2之前，Spring MVC Test framework都是作为一个独立的项目挂在GitHub上面，通过不同人的实际使用、反馈和非常多的贡献来不断的成长。 这个独立的项目目前依然还是可以在Gihub上面找到，并且能在包括 Spring Framework 3.1.x之前的版本中去使用。使用3.2版本以上的应用必须替换依赖spring-test-mvc为spring-test spring-test使用了不一样的包org.springframework.test.web，但是除了两点不同外，其他都是一样的。一点是在3.2版本新增的特性(比如web的异步请求)。另外一点是创建MockMvc实例的一些选项。在3.2以上的版本中，可以只需要通过TestContext框架来做这样的工作，这能够更好的加载配置文件。

Spring MVC Test framework 提供了支持在客户端和服务端侧对Spring MVC 代码进行测试的支持Junit的流畅接口。通常，会通过 TestContext framework 加载实际的Spring配置，并且使用DispatcherServlet 来处理请求。因此可以在近似实际环境的条件下进行测试，而不需要运行一个 Servlet容器。 客户端侧的测试是基于RestTemplate的，允许测试代码是符合RestTemplate的，不需要运行服务去响应请求。

服务端侧的测试

在3.2版本以前的测试中，需要实例化一个controller，注入mock或者基本的依赖，然后直接调用他的方法，这中间必须调用MockHttpServletRequest和MockHttpServletResponse。 虽然这是很容易做到,控制器有很多注释,还有待观察。请求映射、数据绑定类型转换和验证的只是几个例子不是测试。此外,还有其他类型的带注释的方法如@InitBinder @ModelAttribute,@ExceptionHandler得到调用请求处理的一部分。 Spring MVC测试背后的想法是能够重写这些控制器测试通过执行实际的请求和生成响应,就像在运行时,一路上通过Spring MVC DispatcherServlet调用控制器。控制器仍然可以注射模拟依赖关系,所以测试可以在web层保持专注。 Spring MVC测试建立在熟悉的“模拟”Servlet API的实现可以在Spring测试模块。这允许执行请求并生成响应而不需要运行在一个Servlet容器。在大多数情况下都应该是在运行时除了JSP呈现,不可用一个Servlet容器之外。此外,如果您熟悉MockHttpServletResponse是如何工作的,你会知道转发和重定向不实际执行。而不是“转发”和“重定向url保存,可以在测试断言。这意味着如果您使用的是JSP,您可以验证JSP页面的请求被转发。 所有其他的方式呈现包括@ResponseBody方法和视图类型(除了jsp)如Freemarker,Velocity,Thymeleaf,和其他呈现HTML、JSON、XML、等等应该按预期工作,响应将包含生成的内容。 下面是一个例子,一个测试请求JSON格式的帐户信息:

import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.*;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.*;

@RunWith(SpringJUnit4ClassRunner.class)
@WebAppConfiguration
@ContextConfiguration("test-servlet-context.xml")
public class ExampleTests {

    @Autowired
    private WebApplicationContext wac;

    private MockMvc mockMvc;

    @Before
    public void setup() {
        this.mockMvc = MockMvcBuilders.webAppContextSetup(this.wac).build();
    }

    @Test
    public void getAccount() throws Exception {
        this.mockMvc.perform(get("/accounts/1").accept(MediaType.parseMediaType("application/json;charset=UTF-8")))
          .andExpect(status().isOk())
          .andExpect(content().contentType("application/json"))
          .andExpect(jsonPath("$.name").value("Lee"));
    }

}
