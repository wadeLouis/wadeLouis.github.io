1. 基础的配置

想让我们的程序运行在调试模式下，可以这样配置：

1
2

    app = Flask(__name__)
    app.config['DEBUG'] = True

这个布尔值也可以通过Flask对象来设置：

1

    app.debug = True

一般情况下，我们都使用最开始那种方式去配置这个选项。

2. 常见的配置方式

  1. 通过外部文件（*.cfg）等配置 python app.config.from_pyfile('myconfig.cfg')
  2. 通过一个对象来配置，一般这个对象是一个类，当然也可以是模块等等 python app.config.from_object('myapplication.default_settings') # 如果是要加载当前文件的配置，我们可以使用下面的方式 app.config.from_object(__name__) #To load from same file

  3. 通过系统环境变量来配置

    1

        app.config.from_envvar('PATH_TO_CONFIG_FILE')

    在windows中可以这样设置PATH_TO_CONFIG_FILE这个变量：

    1

        set PATH_TO_CONFIG_FILE=c:\\config.cfg

    在linux中可以这样设置：

    1

        export PATH_TO_CONFIG_FILE=c:\\config.cfg

```python

如果要使用bootstrap.min.js，可以这样写：

1

    <script src="/static/js/bootstrap.min.js"></script>

如果存在这样的配置：

1

    app = Flask(__name__, static_url_path='/differentstatic',static_folder='/path/to/static/folder')

那么可以这样使用：

1
2

    <script src="/differentstatic/bootstrap.min.js"></script>

** 平常我们都不是这样用，下面方式才是最推荐使用的：**

1
2

    <script src="{{ url_for('static', filename='js/bootstrap.min.js') }}"></script>

5. instance目录自动配置

如果和应用同级下，存在instance这样一个目录，那么应用将会自动导入instance目录下的配置信息。 可以在配置文件中指定这个目录的地址：

1

    app = Flask(__name__, instance_path='/absolute/path/to/instance/folder')

** 为了加载这个目录下的配置，我们需要设置instance_relative_config **

1

    app = Flask(__name__, instance_relative_config=True)

** 通过这样读取配置 **

1

    app.config.from_pyfile('config.cfg', silent=True)

    在第6节的基础上，我们只要变动两个地方就可以实现：

	from flask import Blueprint # 导入包
	hello = Blueprint('hello', __name__) # 创建蓝图
	## 所有的app对象转换成蓝图对象
	@hello.route('/')
	@hello.route('/hello')
	def hello_world():
		return MESSAGES['default']
	@hello.route('/show/<key>')
	def get_message(key):
		return MESSAGES.get(key) or "%s not found!" % key
	@hello.route('/add/<key>/<message>')
	def add_or_update_message(key, message):
		MESSAGES[key] = message
		return "%s Added/Updated" % key
