Python Nose测试框架命令行详解

2019-01-16
Python 单元测试 nose

文章目录

-V, –version

Output nose version and exit 打印nose的版本号

-p, –plugins

Output list of available plugins and exit. Combine with higher verbosity for greater detail

-v=DEFAULT, –verbose=DEFAULT

Be more verbose. [NOSE_VERBOSE] nose日志输出等级

–verbosity=VERBOSITY

Set verbosity; –verbosity=2 is the same as -v 和上面-v是一样

-q=DEFAULT, –quiet=DEFAULT

Be less verbose 精简的日志输出

-c=FILES, –config=FILES

Load configuration from config file(s). May be specified multiple times; in that case, all config files will be loaded and combined

读取配置文件，可以指定多个，在这种情况下，所有配置文件将被加载和组合

-w=WHERE, –where=WHERE

Look for tests in this directory. May be specified multiple times. The first directory passed will be used as the working directory, in place of the current working directory, which is the default. Others will be added to the list of tests toexecute. [NOSE_WHERE]

在这个目录中查找测试。可以指定多次。通过将第一个目录作为工作目录，在当前工作目录下，这是默认的工作目录。其他人将被添加到测试执行的名单。

–py3where=PY3WHERE

Look for tests in this directory under Python 3.x. Functions the same as ‘where’, but only applies if running under Python 3.x or above. Note that, if present under 3.x, this option completely replaces any directories specified with ‘where’, so the ‘where’ option becomes ineffective. [NOSE_PY3WHERE]

在当前目录下寻找使用python3.x 的测试文件。只支持python3.x以上版本的测试文件。

-m=REGEX, –match=REGEX, –testmatch=REGEX

Files, directories, function names, and class names that match this regular expression are considered tests. Default: (?:^|[b_./-])[Tt]est [NOSE_TESTMATCH]

文件，目录，函数名和匹配这个正则表达式的类名称被认为是测试。正则表达式默认是 (?:^|[b_./-])[Tt]est

–tests=NAMES

Run these tests (comma-separated list). This argument is useful mainly from configuration(配置) files; on the command line, just pass the tests to run as additional(附加的) arguments with no switch.

运行这些测试（用逗号分隔的列表）。这些参数可以很方便的从配置文件中读取。

-l=DEFAULT, –debug=DEFAULT

Activate debug logging for one or more systems. Available debug loggers: nose, nose.importer, nose.inspector, nose.plugins, nose.result and nose.selector. Separate multiple names with a comma.

–debug-log=FILE

Log debug messages to this file (default: sys.stderr)

–logging-config=FILE, –log-config=FILE

Load logging config(配置) from this file – bypasses(旁路) all other logging config settings.

-I=REGEX, –ignore-files=REGEX

Completely ignore(驳回诉讼) any file that matches this regular expression. Takes precedence(优先) over any other settings or plugins(插件). Specifying(指定) this option will replace the default setting. Specify this option multiple times to add more regular expressions [NOSE_IGNORE_FILES]

-e=REGEX, –exclude=REGEX

Don’t run tests that match regular expression [NOSE_EXCLUDE]

-i=REGEX, –include=REGEX

This regular expression will be applied to files, directories, function names, and class names for a chance to include additional(附加的) tests that do not match TESTMATCH. Specify(指定) this option multiple times to add more regular expressions [NOSE_INCLUDE]

-x, –stop

Stop running tests after the first error or failure

-P, –no-path-adjustment

Don’t make any changes to sys.path when loading tests [NOSE_NOPATH]

–exe

Look for tests in python(巨蟒) modules that are executable(可执行的). Normal behavior(行为) is to exclude(排除) executable modules, since they may not be import-safe [NOSE_INCLUDE_EXE]

–noexe

DO NOT look for tests in python(巨蟒) modules that are executable(可执行的). (The default on the windows platform is to do so.)

–traverse-namespace

Traverse(穿过) through all path entries of a namespace package –first-package-wins, –first-pkg-wins, –1st-pkg-wins nose’s importer will normally evict(驱逐) a package from sys.modules if it sees a package with the same name in a different location. Set this option to disable that behavior(行为).

–no-byte-compile

Prevent nose from byte-compiling the source into .pyc files while nose is scanning(扫描) for and running tests.

-a=ATTR, –attr=ATTR

Run only tests that have attributes(属性) specified(指定) by ATTR [NOSE_ATTR]

-A=EXPR, –eval-attr=EXPR

Run only tests for whose attributes the Python expression EXPR evaluates(评价) to True [NOSE_EVAL_ATTR]

-s, –nocapture

Don’t capture(俘获) stdout (any stdout output(输出) will be printed immediately) [NOSE_NOCAPTURE]

–nologcapture

Disable logging capture plugin(插件). Logging configuration(配置) will be left intact(完整的). [NOSE_NOLOGCAPTURE]

–logging-format=FORMAT

Specify(指定) custom format to print statements(声明). Uses the same format as used by standard logging handlers. [NOSE_LOGFORMAT]

–logging-datefmt=FORMAT

Specify custom date/time format to print statements. Uses the same format as used by standard logging handlers. [NOSE_LOGDATEFMT]

–logging-filter=FILTER

Specify which statements to filter in/out. By default, everything is captured(俘获). If the output(输出) is too verbose(冗长的), use this option to filter out needless(不必要的) output. Example: filter=foo will capture statements issued ONLY to foo or foo.what.ever.sub but not foobar or other logger. Specify multiple loggers with comma: filter=foo,bar,baz. If any logger name is prefixed(有前缀的) with a minus, eg filter=-foo, it will be excluded(排除) rather than included. Default: exclude logging messages from nose itself (-nose). [NOSE_LOGFILTER]

–logging-clear-handlers

Clear all other logging handlers

–logging-level=DEFAULT

Set the log level to capture

–with-coverage

Enable plugin(插件) Coverage(覆盖): Activate(刺激) a coverage report using Ned Batchelder’s coverage module. [NOSE_WITH_COVERAGE]

–cover-package=PACKAGE

Restrict coverage(覆盖) output(输出) to selected packages [NOSE_COVER_PACKAGE]

–cover-erase

Erase previously collected coverage statistics(统计) before run

–cover-tests

Include test modules in coverage report [NOSE_COVER_TESTS]

–cover-min-percentage=DEFAULT

Minimum(最小值) percentage of coverage for tests to pass [NOSE_COVER_MIN_PERCENTAGE]

–cover-inclusive

Include all python(巨蟒) files under working directory in coverage report. Useful for discovering holes in test coverage if not all files are imported by the test suite. [NOSE_COVER_INCLUSIVE]

–cover-html

Produce HTML coverage information

–cover-html-dir=DIR

Produce HTML coverage(覆盖) information in dir

–cover-branches

Include branch coverage in coverage report [NOSE_COVER_BRANCHES]

–cover-xml

Produce XML coverage information

–cover-xml-file=FILE

Produce XML coverage information in file

–pdb

Drop into debugger(调试) on failures or errors

–pdb-failures

Drop into debugger on failures

–pdb-errors

Drop into debugger(调试) on errors

–no-deprecated

Disable special handling of DeprecatedTest exceptions(例外).

–with-doctest

Enable plugin(插件) Doctest: Activate(刺激) doctest plugin to find and run doctests in non-test modules. [NOSE_WITH_DOCTEST]

–doctest-tests

Also look for doctests in test modules. Note that classes, methods and functions should have either doctests or non-doctest tests, not both. [NOSE_DOCTEST_TESTS]

–doctest-extension=EXT

Also look for doctests in files with this extension(延长) [NOSE_DOCTEST_EXTENSION]

–doctest-result-variable=VAR

Change the variable(变量的) name set to the result of the last interpreter(解释者) command from the default ‘_’. Can be used to avoid conflicts(冲突) with the _() function used for text translation. [NOSE_DOCTEST_RESULT_VAR]

–doctest-fixtures=SUFFIX

Find fixtures(设备) for a doctest file in module with this name appended(附加) to the base name of the doctest file

–doctest-options=OPTIONS

Specify(指定) options to pass to doctest. Eg. ‘+ELLIPSIS,+NORMALIZE_WHITESPACE’

–with-isolation

Enable plugin(插件) IsolationPlugin: Activate(刺激) the isolation(隔离) plugin to isolate(隔离) changes to external(外部的) modules to a single test module or package. The isolation plugin resets(重新设定) the contents of sys.modules after each test module or package runs to its state before the test. PLEASE NOTE that this plugin should not be used with the coverage(覆盖) plugin, or in any other case where module reloading(重复荷载) may produceundesirable(不良的) side-effects. [NOSE_WITH_ISOLATION]

-d, –detailed-errors, –failure-detail

Add detail to error output(输出) by attempting to evaluate(评价) failed asserts(维护) [NOSE_DETAILED_ERRORS]

–with-profile

Enable plugin Profile: Use this plugin to run tests using the hotshot profiler. [NOSE_WITH_PROFILE]

使用分析器插件：使用这个插件去让测试使用 hotshot profiler

–profile-sort=SORT

Set sort order for profiler output

设置分析器输出的排序顺序

–profile-stats-file=FILE

Profiler stats file; default is a new temp file on each run

分析器统计文件；默认情况下每次测试都是一个新的临时文件

–profile-restrict=RESTRICT

Restrict profiler output. See help for pstats.Stats for details

限制分析器输出。看到帮助pstats.Stats 详情。

–no-skip

Disable special handling of SkipTest exceptions.

禁用skiptest例外特殊处理。

–with-id

Enable plugin TestId: Activate to add a test id (like #1) to each test name output. Activate with –failed to rerun failing tests only. [NOSE_WITH_ID]

启动测试ID插件：自动在输出的时候在测试名称前加上id号。只在使用-failed

–id-file=FILE

Store test ids found in test runs in this file. Default is the file .noseids in the working directory.

保存测试过程中的测试id到文件中。默认的文件是工作目录下的.noseids

–failed

Run the tests that failed in the last test run. 运行上一次测试失败的测试

–processes=NUM

Spread test run among this many processes. Set a number equal to the number of processors or cores in your machine for best results. Pass a negative number to have the number of processes automatically set to the number of cores. Passing 0 means to disable parallel testing. Default is 0 unless NOSE_PROCESSES is set. [NOSE_PROCESSES]

通过多处理器加速测试过程。可以根据你的设备设置多少个核心来测试。负数表示默认使用系统的核心数。0表示不使用。默认是0 除非NOSE_PROCESSES被设置了。

–process-timeout=SECONDS

Set timeout for return of results from each test runner process. Default is 10. [NOSE_PROCESS_TIMEOUT]

为每个测试者进程返回结果设置超时时间。默认为10。

–process-restartworker

If set, will restart each worker process once their tests are done, this helps control memory leaks from killing the system. [NOSE_PROCESS_RESTARTWORKER]

如果配置这一项，那测试完成后将会重启测试工作，这可以帮助保证不因为内存泄露导致系统崩溃

–with-xunit

Enable plugin Xunit: This plugin provides test results in the standard XUnit XML format. [NOSE_WITH_XUNIT]

启用xunit插件：这个插件提供测试结果生成到一个标准的xunit xml文件格式

–xunit-file=FILE Path to xml file to store the xunit report in. Default is nosetests.xml in the working directory [NOSE_XUNIT_FILE] 存 储xunit 报告的的xml路径。默认是在工作目录下的nosetests.xml

Name of the testsuite in the xunit xml, generated by plugin. Default test suite name is nosetests.

在xunit.xml中配置的测试集的名称，这个是通过插件生成的。默认的测试集是nosetests

–all-modules

Enable plugin AllModules: Collect tests from all python modules. [NOSE_ALL_MODULES]

启用所有模块的测试：收集所有python的模块

–collect-only

Enable collect-only: Collect and output test names only, don’t run any tests. [COLLECT_ONLY]

只收集和输出测试的名称
